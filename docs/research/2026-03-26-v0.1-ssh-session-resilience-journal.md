## Problem Statement

Greg connects to his Proxmox Claude Code server via SSH from Windows 11. SSH disconnections kill the Claude session. Additionally, multiple developers need to collaborate on the same Claude session for education, supervision, and shared development.

## Direction Confirmed: PTY Multiplexer with SSH Server

### Key Architectural Decision
**Wrap `./claude` CLI via PTY, not Agent SDK.** The proxy is a terminal multiplexer, not a Claude reimplementation. This preserves all hooks, plugins, skills, /commands, and the full Claude Code UX for free.

### Why not Agent SDK?
- Would have to reimplement everything Claude Code already does
- Hooks, plugins, skills, /commands — none of it works through SDK
- CLI updates just work; SDK would require maintenance
- Session resume is built-in with `--resume`

### User Context
- Multiple collaborators joining same stream
- Education/supervision use case
- Dead simple: SSH to box, it works
- Sessions run as configurable system user (`su -u <user> claude`)

## Bugs & Lessons Learned

### Nested Quote Escaping in tmux + su (2026-03-27)
**Problem:** When launching Claude via tmux with `su - user -c 'command args'`, the tmux `new-session` command also wraps the command in single quotes: `tmux new-session -d -s name 'su - user -c '...'`. Nested single quotes break shell parsing silently — tmux creates the session but the command fails immediately with exit code 1.

**Impact:** Resume sessions (`claude --resume <id>`) died instantly because the `--resume` args inside the `su -c` created triple-nested quotes. New sessions worked by accident because the simple case (`su - greg -c '/path/to/launcher.sh'`) had no nested quotes.

**Fix:** Write the inner command to a temporary bash script, have tmux execute the script. The script self-deletes on startup (`rm -f "$0"` equivalent). Eliminates all quoting issues regardless of argument complexity.

**Lesson:** Whenever you're building shell commands by string concatenation across multiple layers (node → tmux → su → bash → command), use temp scripts instead of trying to escape quotes. Each layer has its own quoting rules and they compound.

### Alternate Screen vs Native Scrolling (2026-03-26)
**Problem:** Using `\e[?1049h` (alternate screen) for clean status bar rendering disables PuTTY's native scrollback — by design. Users couldn't scroll up to see previous output.

**Fix:** Removed alternate screen entirely. Status info moved to PuTTY title bar via OSC escape (`\e]0;title\x07`). Native scrolling works. Status bar will return in Phase 3 web UI where we control the renderer.

**Lesson:** Don't fight the terminal. Alternate screen and scroll regions are for full-screen apps (vim, less, tmux). If you want native scrollback, you can't use them. The title bar is free real estate that doesn't interfere with anything.

### xterm-headless async write (2026-03-27)
**Problem:** `@xterm/headless` Terminal.write() is async — the buffer isn't populated until a callback fires. Reading the buffer immediately after write returns empty.

**Fix:** Flush with `vterm.write('', () => { /* read buffer here */ })` before reading.

## Design Decisions Log

1. **Full proxy with web UI (phased)** — not just tmux, not just hooks
2. **SSH server as transport** — `ssh -p 3100 droidware`, keys for auth
3. **Free-for-all input Phase 1** → role-based Phase 2
4. **Lobby on connect** — list active sessions, create new, join existing
5. **PTY wrapper, not Agent SDK** — multiplexes the real `claude` process
6. **Configurable run-as user** — `su -u <user> claude` for per-project/team isolation
